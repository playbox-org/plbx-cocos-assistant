# Playable Developer Guide

How to set up a Cocos Creator playable project so all ad-network validators detect the right lifecycle events (Moloco V2, Mintegral PlayTurbo, AppLovin, IronSource, etc.).

One adapter (`plbx_html_playable.ts`) wires every supported network. You write code once, the packager handles per-network differences.

---

## Quick start (3 minutes)

### 1. Install the extension

Copy or symlink this extension into your Cocos Creator project's `extensions/` directory, then enable it in the Cocos Extension Manager.

### 2. Generate the adapter

In Cocos Creator → open **Playbox** panel → click **Generate adapter**.

Creates `assets/Scripts/plbx_html/plbx_html_playable.ts`. Do not edit this file — it's regenerated on demand.

### 3. Wire up the lifecycle

Add **4 calls** to your game code (see [Lifecycle methods](#lifecycle-methods) below):

| Call | Where |
|---|---|
| `plbx.game_ready()` | Once when main scene is loaded |
| `plbx.tap()` | Every user touch / interaction |
| `plbx.download()` | CTA / DOWNLOAD button click |
| `plbx.game_end()` | When gameplay ends |

That's it. Now package via **Playbox panel → Package → choose networks → Build**.

## Lifecycle methods

Import once per file that needs them:

```typescript
import plbx from '../plbx_html/plbx_html_playable';
```

### `plbx.game_ready()`

**When:** game finished loading, scene visible, ready to play.

**Fires:** `game_viewable` beacon (Moloco V2), `gameReady` signal (Mintegral PlayTurbo, TikTok/Pangle playableSDK).

**Where to call:** main scene `onLoad()` or `start()`, after `initializeSystems()`.

```typescript
protected onLoad(): void {
  this.initializeSystems();
  plbx.game_ready();
}
```

⚠️ Call exactly **once** per session. Adapter dedupes internally but explicit single-call is cleaner.

---

### `plbx.tap()`

**When:** every meaningful user tap / touch / click.

**Fires:** `engagement` beacon at `taps_for_engagement` threshold (default 1), `redirection` beacon at `taps_for_redirection` threshold (default 3). Moloco DSP overrides defaults per campaign.

**Where to call:** central input handler. For Cocos this is usually `TOUCH_START`:

```typescript
import { input, Input, EventTouch } from 'cc';

input.on(Input.EventType.TOUCH_START, (event: EventTouch) => {
  plbx.tap();
}, this);
```

Or inside `InputSystem.onTouchStart` if you have one:

```typescript
private onTouchStart(event: EventTouch): void {
  plbx.tap();
  // ... existing logic
}
```

Safe to call hundreds of times — `engagement` / `redirection` fire idempotently (once each per session).

---

### `plbx.download()`

**When:** user pressed the CTA / DOWNLOAD button.

**Fires:** `click` beacon, opens `mraid.open(final_url)` (Moloco V2) or network-equivalent CTA path.

**Where to call:** CTA button click handler.

```typescript
@property(Button)
ctaButton: Button = null;

protected onLoad(): void {
  this.ctaButton.node.on(Button.EventType.CLICK, () => {
    plbx.download();
  }, this);
}
```

Multiple presses → multiple `click` beacons (each is a separate billable click). Adapter applies a 100ms debounce per beacon to filter double-fire bugs from Cocos input layers.

---

### `plbx.game_end()`

**When:** gameplay session is over — level finished, time ran out, user reached fail/success screen.

**Fires:** `complete` beacon (Moloco V2), `gameEnd` signal (Mintegral, Vungle).

**Where to call:** end-of-game flow. For Roadside Empire pattern it's `CTAView.show()`:

```typescript
public show(): void {
  this.node.active = true;
  plbx.game_end();
  // ... animation
}
```

Fires idempotently — multiple calls produce one `complete` beacon per session.

---

### `plbx.is_muted()` *(optional but recommended)*

**When:** before initializing AudioContext / playing music.

**Returns:** `true` if ad container requested muted playback at start (Moloco `MOLOCO_MACROS.start_muted`, etc.).

```typescript
initAudio(): void {
  if (plbx.is_muted()) {
    return; // do not start audio
  }
  this.musicSource.play();
}
```

---

### `plbx.report(eventKey: string)` *(optional)*

**When:** custom telemetry beyond standard lifecycle.

**Fires:** matching `MOLOCO_MACROS[eventKey]` beacon. No-op on non-Moloco networks.

```typescript
onTutorialCompleted(): void {
  plbx.report('tutorial_complete');
}
```

Only works if the network's macro map contains your custom key. Standard lifecycle (`game_ready`, `tap`, `download`, `game_end`) is preferred.

## Full Cocos Creator example

`GameEntryPoint.ts`:
```typescript
import { _decorator, Component, Button } from 'cc';
import plbx from '../plbx_html/plbx_html_playable';

const { ccclass, property } = _decorator;

@ccclass('GameEntryPoint')
export class GameEntryPoint extends Component {
  @property(Button) ctaButton: Button = null;

  protected onLoad(): void {
    plbx.set_app_store_url('https://apps.apple.com/.../id123');
    plbx.set_google_play_url('https://play.google.com/store/apps/details?id=...');

    this.initializeSystems();

    this.ctaButton.node.on(Button.EventType.CLICK, () => {
      plbx.download();
    }, this);

    plbx.game_ready();
  }

  private initializeSystems(): void {
    // ... your systems
  }
}
```

`InputSystem.ts` (or wherever touch is handled):
```typescript
import { input, Input, EventTouch } from 'cc';
import plbx from '../plbx_html/plbx_html_playable';

export class InputSystem {
  constructor() {
    input.on(Input.EventType.TOUCH_START, this.onTouchStart, this);
  }

  private onTouchStart(event: EventTouch): void {
    plbx.tap();
    // ... your input logic
  }
}
```

`CTAView.ts` (end-of-game screen):
```typescript
import { _decorator, Component, Button } from 'cc';
import plbx from '../plbx_html/plbx_html_playable';

const { ccclass, property } = _decorator;

@ccclass('CTAView')
export class CTAView extends Component {
  @property(Button) storeButton: Button = null;

  public show(): void {
    this.node.active = true;
    plbx.game_end();

    this.storeButton.node.on(Button.EventType.CLICK, () => {
      plbx.download();
    }, this);
  }
}
```

## Validation workflow

### 1. Build the Cocos web-mobile output

Cocos Editor → **Project → Build** → platform `web-mobile`. Produces `build/web-mobile/`.

### 2. Package for ad networks

Playbox panel → **Package** tab → check the networks you target (e.g. `molocoV2`, `applovin`, `mintegral`) → click **Build**.

Outputs in `build/plbx-html/{networkId}/`.

### 3. Open Preview Validator

Playbox panel → **Preview** button (next to Build). Opens a local browser window with per-network tabs.

### 4. Run smoke test (per network tab)

| Check | How to trigger |
|---|---|
| File size | auto |
| MRAID ready | auto |
| viewableChange listener registered | auto (defer-boot gate registers) |
| `mraid_viewable` beacon | click **Viewable** manual trigger |
| `game_viewable` beacon | auto (Cocos boot calls `plbx.game_ready()`) |
| `click` beacon | tap CTA button in playable OR **CTA** manual trigger |
| `engagement` beacon | tap canvas 1× OR **Simulate taps** N=1 |
| `redirection` beacon | tap canvas 3× OR **Simulate taps** N=3 |
| `complete` beacon | play through to end OR **End game** manual trigger |
| `final_url` consumed | auto when CTA fires `mraid.open(final_url)` |

All green ✅ → playable is ready for Moloco QA submission.

## Network behavior reference

Same code, different beacon paths per network:

| Network | `download()` path | `game_ready()` path | `game_end()` path | `tap()` |
|---|---|---|---|---|
| Moloco V2 | `mraid.open(final_url)` + `click` macro | `game_viewable` macro | `complete` macro | engagement / redirection thresholds |
| AppLovin / Unity / IronSource | `mraid.open(storeUrl)` | gameReady signal | gameEnd signal | (no-op) |
| Facebook / Moloco (legacy) | `FbPlayableAd.onCTAClick()` | (no-op) | gameEnd signal | (no-op) |
| Mintegral | `window.install()` | `gameReady()` | `gameEnd()` | (no-op) |
| TikTok / Pangle | `playableSDK.openAppStore()` | `playableSDK.reportGameReady()` | `playableSDK.reportGameClose()` | (no-op) |
| Google | `ExitApi.exit()` | (no-op) | (no-op) | (no-op) |

You don't write network-specific code. Adapter abstracts it.

## Troubleshooting

**`game_viewable` not detected (30s timeout)**
→ `plbx.game_ready()` not called. Add it to your main scene `onLoad`.

**`mraid_viewable` not detected**
→ Click the **Viewable** manual trigger in preview validator. In production, the ad container fires this automatically when it becomes visible. If your defer-boot gate is missing, game won't boot — see required string check in packager output.

**`engagement` / `redirection` not detected**
→ `plbx.tap()` not wired to touch handler. Add to your input system's `TOUCH_START` callback.

**`complete` not detected**
→ `plbx.game_end()` not called on level completion. Add to end-of-game flow (success/fail screen show).

**`click` beacon fires 2× per CTA press**
→ Two listeners hitting `download()` simultaneously. Adapter has a 100ms debounce that catches this. If you still see it: check for duplicate event bindings in your CTA button code.

**Cocos game shows black screen in production but works in preview**
→ MRAID defer-boot gate issue. Verify `__plbx_pre_boot` is in the final HTML (validator check `viewableChange listener registered` should be green).

**File size over network limit**
→ Compress textures (TinyPNG / use the Playbox panel's compression tools), reduce audio bitrate, remove unused assets. Most networks cap at 5 MB.

**`is_muted()` always returns false in preview**
→ Expected. Preview's `MOLOCO_MACROS.start_muted` is the placeholder string. Production gets a real `"1"` or `"0"` from DSP.

## What NOT to do

- ❌ Don't call `mraid.open()` / `FbPlayableAd.onCTAClick()` / `window.install()` directly — use `plbx.download()`
- ❌ Don't read `MOLOCO_MACROS` directly — use `plbx.is_muted()` etc.
- ❌ Don't define `window.gameReady` / `window.gameStart` / `window.gameClose` manually — runtime-loader handles these
- ❌ Don't load external trackers (Google Analytics, Facebook Pixel, etc.) — fails validator checks
- ❌ Don't ship the `super_html_playable.ts` legacy adapter — use `plbx_html_playable.ts` only

## Cheat sheet

```typescript
// Once per project: import the adapter where needed
import plbx from '../plbx_html/plbx_html_playable';

// Main scene onLoad
plbx.set_app_store_url('https://apps.apple.com/.../id123');
plbx.set_google_play_url('https://play.google.com/store/apps/details?id=...');
plbx.game_ready();

// CTA button
plbx.download();

// Every touch
plbx.tap();

// End of game
plbx.game_end();

// Optional: audio
if (plbx.is_muted()) { /* skip music */ }
```

Wire those 6 lines correctly and every network validator should pass.
